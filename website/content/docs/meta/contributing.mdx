## How to Contribute

We welcome contributions! Here's how to get started.

## Development Setup

### Fork and Clone

```bash
git clone https://github.com/nesalia-inc/better-notion.git
cd better-notion
```

### Install Development Dependencies

```bash
uv sync
# or
pip install -e ".[dev]"
```

### Run Tests

```bash
# All tests
uv run pytest

# Unit tests only
uv run pytest tests/unit/

# With coverage
uv run pytest --cov=better_notion
```

## Code Style

### Formatting

```bash
uv run ruff format .
```

### Linting

```bash
uv run ruff check .
```

## Making Changes

### Branch Naming

- `feature/` - New features
- `fix/` - Bug fixes
- `docs/` - Documentation changes
- `test/` - Test updates

### Commit Messages

Follow conventional commits:

```
feat: add new feature
fix: correct bug in page update
docs: update API reference
test: add integration tests
```

## Submitting Changes

1. Create feature branch
2. Make your changes
3. Add tests for new features
4. Ensure tests pass
5. Submit pull request

## Code Review

All contributions go through code review. We'll check for:
- Code quality and style
- Test coverage
- Documentation updates
- Breaking changes
