Welcome to the API Reference section. This contains complete technical documentation for all SDK components.

## Overview

The Better Notion SDK is organized into:

- **Client** - Main API client (`NotionAPI`)
- **Entities** - Page, Block, Database, User objects
- **Collections** - Managers for groups of entities
- **Properties** - Type-safe property builders
- **Exceptions** - Error types and handling

## Hierarchy

```
NotionAPI (client)
├── pages: PageCollection
├── blocks: BlockCollection
├── databases: DatabaseCollection
└── users: UserCollection

Entities:
├── Page
├── Block
├── Database
└── User

Properties:
├── Title
├── Text / RichText
├── Select / MultiSelect
├── Date
├── Checkbox
├── Number
└── URL / Email / Phone

Exceptions:
└── NotionAPIError (base)
    ├── HTTPError
    ├── ValidationError
    └── ConfigurationError
```

## Common Patterns

### Initialize Client

```python
from better_notion import NotionAPI

api = NotionAPI(auth="secret_...")
```

### Access Collections

```python
# Page operations
page = await api.pages.get("page_id")

# Block operations
blocks = await api.blocks.children("block_id")

# Database queries
result = await api.databases.query("database_id")

# User operations
user = await api.users.me()
```

### Use Entities

```python
# Get page entity
page = await api.pages.get("page_id")

# Update entity
await page.update(**Select("Status", "Done").build())

# Save changes
await page.save()
```

### Handle Errors

```python
from better_notion._api.errors import NotFoundError

try:
    page = await api.pages.get("page_id")
except NotFoundError:
    print("Page not found")
```

## Type Hints

